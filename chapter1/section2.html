关系型数据库

如何防范关系型数据库Sql注入

常见的安全编码方式：预编译+输入验证 预编译适用于大多数对数据库操作的场景

错误写法：直接拼接用户输入的参数

jdbc:

Statement statement = con.createStatement();
String sql = "select * from users where username = '" + username + "'";
ResultSet rs = statement.executeQuery(sql);

mybatis:

@Select("select * from users where username = '${username}'")
List<User> findByUserNameVuln01(@Param("username") String username);

正确写法：使用预编译方式

jdbc:

String sql = "select * from users where username = ?";
PreparedStatement st = con.prepareStatement(sql);

mybatis:

@Select("select * from users where username = #{username}")
User findByUserName(@Param("username") String username);

但预编译并不是万能的，涉及到查询参数里使用表名、字段名的场景时(如order by、limit、group by等)，不能使用预编译，因为会产生语法错误。 在无法使用预编译的场景，可以使用数据校验的方式来拦截非法参数，数据校验推荐使用白名单方式。

不能使用预编译的正确写法：

public String Sqlfilter(String sort){
    return Pattern.matches("[A-Za-z0-9_]+",sort) ? sort : "unauthorized sort";
}
